Anti-AI Rules

Enforced

These rules are baked into every Gemini prompt to keep posts sounding human.

  • No em dashes (—) ever
  • No "Game-changer", "Leverage", "Seamless"
  • No "In conclusion" or "In summary"
  • No "I am excited to share"
  • No perfect bullet-point list structure
  • No more than 5 hashtags
  • Specific numbers and real details
  • Emotional moments and vulnerability
  • Mix of short punchy + longer sentences
  • First-person voice throughout

Setup Guide

One-time setup
1

Register LinkedIn Developer App

Visit the LinkedIn Developer Portal and create a new app.

Open LinkedIn Developer Portal →
  • App name: "LinkedIn Auto-Post" (or anything)
  • Associate with your LinkedIn company page (or create one)
  • Go to "Products" tab → Request access to "Share on LinkedIn"
  • Under "Auth" tab → Add redirect URL: http://localhost:8765/callback
  • Copy your Client ID and Client Secret
2

Get Gemini API Key

Free tier — no credit card needed.

Open Google AI Studio →
  • Sign in with Google
  • Click "Create API Key"
  • Copy the key
3

Run token_helper.py locally

This exchanges your LinkedIn credentials for OAuth tokens.

# Create a .env file first: LINKEDIN_CLIENT_ID=your_client_id LINKEDIN_CLIENT_SECRET=your_client_secret # Then run: pip install requests python-dotenv python scripts/token_helper.py

This opens LinkedIn login in your browser, then prints your tokens. Copy them all.

4

Create GitHub PAT

Create Token →
  • Token name: "LinkedIn Auto-Post"
  • Expiration: No expiration (or 1 year)
  • Scopes: Check repo (full) and workflow
  • Click "Generate token" and copy it
5

Add GitHub Repository Secrets

Go to your repo → Settings → Secrets and variables → Actions → New repository secret

Secret Name Value
GEMINI_API_KEY Your Google AI Studio key
LINKEDIN_CLIENT_ID From LinkedIn Developer Portal
LINKEDIN_CLIENT_SECRET From LinkedIn Developer Portal
LINKEDIN_ACCESS_TOKEN From token_helper.py output
LINKEDIN_REFRESH_TOKEN From token_helper.py output
LINKEDIN_USER_URN From token_helper.py output
GH_PAT Your GitHub Personal Access Token
6

Push to GitHub and Enable Pages

git init git add . git commit -m "Initial commit" git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git git push -u origin main
  • Go to repo Settings → Pages
  • Source: Deploy from branch → main → /dashboard folder
  • Your dashboard is now live at: https://username.github.io/repo-name
✓

Done! Posts will start automatically.

GitHub Actions will run at 12:00, 17:00, and 23:00 Pakistan time every day. You can also use the "Manual Post" tab to post immediately.

Note: LinkedIn's "Share on LinkedIn" product approval may take 1-3 days after requesting it. The workflow will fail until it's approved.
